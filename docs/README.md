## Documentation Standards

All Smith documentation follows these principles:

- **Professional Tone**: Technical and precise language
- **Code Examples**: Practical, tested examples
- **Up-to-Date**: Synchronized with codebase
- **Comprehensive**: Covers all features and use cases
- **Accessible**: Clear structure and navigation

## Contributing to Documentation

Documentation improvements are welcome. See [Contributing Guidelines](contributing.md) for details.

- Use proper markdown formatting
